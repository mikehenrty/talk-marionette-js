Whare are Marionette JS tests?

  • Tests for user interactions (tapping, swiping)
  • Can test backed APIs, but those are better for unit tests
  • Mostly runs on mulet, but also runs on device

What are Marionette JS tests not?

  • Not for testing ugli-ness
  • Not for testing functions
  • Not for testing code modules
  • Not bulletproof, racy

$ make test-integration

  • downloads latest mulet (mozilla-download)
  • creates configurable profile (profile builder)
  • runs mulet with configured profile
  • runs a client process that connects to Marionette in mulet
  • client runs mocha tests using Marionette commands
